# Printerceptor Modular

This project automatically detects, extracts, and post-processes text files generated by the **clawPDF** virtual printer.

## Project Structure
- `printerceptor/`: Core logic package.
  - `app.py`: Orchestrates the main thread (GUI) and background thread (Watcher).
  - `config.py`: Local paths and directory setup.
  - `processor.py`: Robust text reading and PDF generation logic.
  - `gui.py`: The fuzzy-search selection overlay.
  - `win32_utils.py`: Low-level Windows focus stealing hacks.
- `main.py`: The entry point for the application.
- `customers.json`: Your customer database.
- `example*.py`: Minimal reference files for focus-stealing tests.

## How it works
1. **Print to clawPDF**: You print a document.
2. **Auto-Save**: clawPDF outputs a `.txt` into the `claw/` folder.
3. **Detection**: The background watcher detects the file and triggers the GUI.
4. **Overlay**: A topmost window appears immediately with keyboard focus.
5. **Selection**: Type to find a customer (fuzzy search) and press Enter.
6. **Output**: A timestamped, headered PDF is generated in `output_pdfs/`.

---

## Usage
Start the modular application:
```powershell
.\.venv\Scripts\python.exe main.py
```

## Dependencies
Ensure you have the virtual environment ready:
```powershell
pip install -r requirements.txt
```
Current requirements: `watchdog`, `fpdf2`, `rapidfuzz`, `pywin32`.
