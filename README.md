# clawPDF Text Watcher

This project automatically detects, extracts, and post-processes text files generated by the **clawPDF** virtual printer.

## How it works

1.  **Print to clawPDF**: You print a document from any application.
2.  **Auto-Save to `claw/`**: `clawPDF` is configured to output a `.txt` file directly into the `claw/` directory of this project.
3.  **Watcher Interception**: The `watcher.py` script detects the new file immediately.
4.  **Extraction & PDF Reform**: It reads the text, applies your post-processing, and generates a clean PDF in `output_pdfs/`.
5.  **Archive**: The original text file is moved to `archive/` for safekeeping.

---

## Setup Instructions

### 1. Install Dependencies
You need `clawPDF` installed on your Windows 11 system:
```powershell
winget install clawSoft.clawPDF
```

### 2. Configure clawPDF Profile
Open **clawPDF Settings** and create or modify a profile:
- **Output Format**: Plain Text (.txt)
- **Auto-save**: Enabled
- **Auto-save directory**: `D:\Users\florian\repos_private\maxtouch_printer\claw`
- **Filename template**: `<JobName>` (or whatever you prefer)

### 3. Initialize Python Environment (Optional if already setup)
If you haven't set up the venv yet:
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

---

## Usage

### Run the Watcher
Start the script using the local virtual environment:
```powershell
.\.venv\Scripts\python.exe watcher.py
```

Any new `.txt` file arriving in `claw/` will now be automatically processed.

### Project Folders
- `claw/`: Input folder (where clawPDF saves text files).
- `output_pdfs/`: Where the cleaned/re-rendered PDFs are stored.
- `archive/`: Original intercepted text files.
- `.venv/`: Local Python environment.
